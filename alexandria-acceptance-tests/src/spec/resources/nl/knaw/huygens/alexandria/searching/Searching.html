Given there are 8 annotated resources in the database owned by user nederlab, When we /searches the following body { "query": { "where": "who:eq(\"nederlab\")", "pageSize": 3 } } we expect status 201 Created, an response body, and a https://{host}/searches/ Location header.
In this case the generated search id (taken from the Location header) is: .
When we /searches/$searchId we expect status 200 OK and a response body containing: (only paging details shown for brevity) { "searchResult" : { "totalResults" : 8, "totalResultPages" : 3, "pageSize" : 3, "^firstResultPage" : "https://{host}/searches/$searchId/resultPages/1" } }
When we /searches/$searchId/resultPages/1 we expect status 200 OK and body (showing only details relevant to paging, actual record details withheld for brevity) { "searchResultPage" : { "pageNumber" : 1, "records" : [ { "_resultNumber" : 1 }, { "_resultNumber" : 2 }, { "_resultNumber" : 3 } ], "^nextPage" : "https://{host}/searches/$searchId/resultPages/2" } }
A convenience link ^nextPage is included on the first page.
Now /searches/$searchId/resultPages/2 results in status 200 OK and body { "searchResultPage" : { "pageNumber" : 2, "records" : [ { "_resultNumber" : 4 }, { "_resultNumber" : 5 }, { "_resultNumber" : 6 } ], "^nextPage" : "https://{host}/searches/$searchId/resultPages/3", "^previousPage" : "https://{host}/searches/$searchId/resultPages/1" } }
Both ^nextPage and ^previousPage convenience links are included on the middle pages.
And /searches/$searchId/resultPages/3 results in status 200 OK and body { "searchResultPage" : { "pageNumber" : 3, "records" : [ { "_resultNumber" : 7 }, { "_resultNumber" : 8 } ], "^previousPage" : "https://{host}/searches/$searchId/resultPages/2" } }
The convenience link ^nextPage is now absent, signaling this is the final page.
When we try to /searches/$searchId/resultPages/0 we receive 404 Not Found Likewise /searches/$searchId/resultPages/4 results in status 404 Not Found as the requested page lies beyond the maximum.
Valid result page numbers are 1 up to and including the number of totalResultPages.
We can successfully /searches the following body { "query": { "find": "annotation", "where": "who:eq(\"nederlab\")", "sort": "-when", "return": "id,resource.url", "pageSize": 10 } } where the properties of query have the following semantics:
find
the Alexandria concept to look for. (currently annotation is the default as well as the only valid value)
where
(mandatory) used to filter records. Check the ANTLR v4 grammar file for syntax, field names and functions. (caveat: multiple where clauses are treated as a conjunction, meaning ALL clauses have to match)
sort
used to sort the result-set on a field (ref. where). Use + for ascending and - for descending order. (if left unspecified, -when is used by default)
return
used to select which fields (ref. where) from the result-set are returned in the output. (if left unspecified, id is returned by default)
pageSize
how many results from the result-set to include in each page of results. (if left unspecified, 10 results per page are included by default)
