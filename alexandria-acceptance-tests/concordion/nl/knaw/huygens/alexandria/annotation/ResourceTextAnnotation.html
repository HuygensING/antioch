How do I annotate a part of my resource text?
Given I have successfully /resources/599fad4f-21ca-4488-94f7-66fba8b39aba the following body: 
{
  "resource" : {
    "id" : "599fad4f-21ca-4488-94f7-66fba8b39aba",
    "ref" : "Letter from Clusius to Lipsius, 1597-01-01"
  }
}
When I /resources/599fad4f-21ca-4488-94f7-66fba8b39aba/text the body: 
{
  "text" : {
    "body" : "<text><p xml:id=\"p0\">paragraph 1</p><p>paragraph 2</p></text>"
  }
}
Then I should receive:
  • status 204 No Content
  • an   response body
To add an annotation on the first paragraph, when I /resources/599fad4f-21ca-4488-94f7-66fba8b39aba/annotations the following body: 
{
  "annotation" : {
    "locator" : "id:p0",
    "type" : "language",
    "value" : "en"
  }
}
then I should receive:
  • status 201 Created
  • a Location header starting with https://{host}/annotations/ and ending in a well-formed UUID
  • an   response body
What happens when I use an invalid locator prefix?
Currently, the only locator prefix implemented is "id", for locating by xml:id.
When I /resources/599fad4f-21ca-4488-94f7-66fba8b39aba/annotations the following body: 
{
  "annotation" : {
    "locator" : "illegalPrefix:something.or.other",
    "type" : "note",
    "value" : "does this work?"
  }
}
then I should receive:
  • status 400 Bad Request
  • and body 
    {
  "error" : {
    "message" : "The locator prefix 'illegalPrefix' is not a valid prefix. Valid prefix: 'id'."
  }
}
What happens when the locator is not valid for the resource text?
The locator given in the annotation body should map to exactly one element in the resource text:
When I /resources/599fad4f-21ca-4488-94f7-66fba8b39aba/annotations the following body: 
{
  "annotation" : {
    "locator" : "id:p100",
    "type" : "something",
    "value" : "something"
  }
}
then I should receive:
  • status 400 Bad Request
  • and body 
    {
  "error" : {
    "message" : "The resource text has no element with xml:id=\"p100\""
  }
}
