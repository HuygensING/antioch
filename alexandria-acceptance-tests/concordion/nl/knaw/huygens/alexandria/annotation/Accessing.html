How can I retrieve annotations?
Annotations can be found both via GET on the object they are annotating, or via the /searches endpoint using a query. Here we show how the state of an annotation influences when it shows up. Annotations are always in one of four states:
TENTATIVE
The annotation has just been created, but is not yet confirmed
CONFIRMED
The annotation has been confirmed and is fully operational
DEPRECATED
This revision of the annotation has been deprecated by a newer revision
DELETED
The annotation is considered deleted (but is still available for review)
Assume a resource exists with uuid: 6cf02b7a-7244-11e5-a68b-ab226ffa3019
  1. We add an annotation via /resources/6cf02b7a-7244-11e5-a68b-ab226ffa3019/annotations 
    {
  "annotation" : {
    "value" : "a tentative annotation"
  }
}
    which gets id da79692b-2711-4e6f-985e-aeae0bdc4f4e
    We leave this annotation in its TENTATIVE state.
  2. We add an annotation via /resources/6cf02b7a-7244-11e5-a68b-ab226ffa3019/annotations 
    {
  "annotation" : {
    "value" : "a confirmed annotation"
  }
}
    which gets id e461e9d0-50a7-45a4-85b1-4afda5246fd2
    We confirm it /annotations/e461e9d0-50a7-45a4-85b1-4afda5246fd2/state 
    {
  "state" : "CONFIRMED"
}
  3. We add an annotation via /resources/6cf02b7a-7244-11e5-a68b-ab226ffa3019/annotations 
    {
  "annotation" : {
    "value" : "a deleted-before-confirmation annotation"
  }
}
    which gets id 2d4e8ef1-07ca-4788-ac08-b0d1ec08f0c8
    We then delete it via /annotations/2d4e8ef1-07ca-4788-ac08-b0d1ec08f0c8 before having confirmed it.
    A TENTATIVE annotation that is deleted in this way will be removed from the database. It will not show up when we GET the resource later on.
  4. We add an annotation via /resources/6cf02b7a-7244-11e5-a68b-ab226ffa3019/annotations 
    {
  "annotation" : {
    "value" : "a deleted-after-confirmation annotation"
  }
}
    which gets id 567cb2be-1bd6-421e-8155-f2672350560c
    We confirm it: /annotations/567cb2be-1bd6-421e-8155-f2672350560c/state 
    {
  "state" : "CONFIRMED"
}
    We then delete it via /annotations/567cb2be-1bd6-421e-8155-f2672350560c
  5. We add an annotation via /resources/6cf02b7a-7244-11e5-a68b-ab226ffa3019/annotations 
    {
  "annotation" : {
    "value" : "an annotation to be deprecated"
  }
}
    which gets id 22104536-2dc9-453c-a0de-4a17737fa626
    We confirm it: /annotations/22104536-2dc9-453c-a0de-4a17737fa626/state 
    {
  "state" : "CONFIRMED"
}
    We deprecate it: /annotations/22104536-2dc9-453c-a0de-4a17737fa626 
    {
  "annotation" : {
    "value" : "the annotation value after deprecation"
  }
}
When we /resources/6cf02b7a-7244-11e5-a68b-ab226ffa3019 we expect 200 OK and body 
{
  "resource" : {
    "id" : "6cf02b7a-7244-11e5-a68b-ab226ffa3019",
    "^annotations" : [ "https://{host}/annotations/da79692b-2711-4e6f-985e-aeae0bdc4f4e", "https://{host}/annotations/e461e9d0-50a7-45a4-85b1-4afda5246fd2", "https://{host}/annotations/567cb2be-1bd6-421e-8155-f2672350560c", "https://{host}/annotations/22104536-2dc9-453c-a0de-4a17737fa626" ]
