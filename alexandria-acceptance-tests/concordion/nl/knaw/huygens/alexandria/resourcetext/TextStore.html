1) How do I submit a text as XML and retrieve it as XML?
Given that I have successfully /resources/22222222-2222-2222-2222-222222222222 the body:

  }
}

When I /resources/22222222-2222-2222-2222-222222222222/text the body: 
<text>
  <p xml:id="p-1" lang="en" auth="me">paragraph 1</p>
  <p>Párâgràph 2</p>
</text>
using text/xml,
then I should receive
  • status 202 Accepted
  • location header https://{host}/resources/22222222-2222-2222-2222-222222222222/text/status
  • and the response body is  

When I /resources/22222222-2222-2222-2222-222222222222/text/status then I should receive
  • status 200 OK
  • and body (only relevant fields shown for brevity) 
    {
  "textImportStatus" : {
    "done" : true,
    "state" : "done",
    "processingTime" : "{duration.anyValid}",
    "validationErrors" : [ ],
    "startTime" : "{date.beforeNow}",
    "expires" : "{date.afterNow}"
  }
}
If the status indicates the text is still being processed (done=false, state=processing), we should wait a while before getting the status again.

Since textImportStatus.done == true, and there are no validationErrors,
/resources/22222222-2222-2222-2222-222222222222/text/xml
using text/xml
yields response body:
<text>
  <p xml:id="p-1" lang="en" auth="me">paragraph 1</p>
  <p>Párâgràph 2</p>
</text>
(the originally uploaded text) (formatting added for increased readability)

3) What if I upload a text to a resource that already has a text?

  • status 409 Conflict
  • and body

4) How do I get a representation of the textgraph in DOT?
Given the resource I made earlier, when I /resources/33333333-3333-3333-3333-333333333333/text/dot using text/plain, then I should receive
  • status 200 OK
  • and body 
    digraph TextGraph {
  ranksep=1.0

  t0 [shape=box, label="different␣text"];
  a0 [label="text"];
  a1 [label="p"];
  a0 -> a1;
  a1 -> t0 [color="blue"];

  {rank=same;t0;}
  {rank=same;a0;}
  {rank=same;a1;}
}
    (notice how spaces in the text have been replaced by ␣ for clarity)
