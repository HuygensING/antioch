What happens when I GET a non-existent resource?
Given that no resource exists with ID 3ed4faaa-c0cd-11e4-a84e-83ef41cbdc44
When I /resources/3ed4faaa-c0cd-11e4-a84e-83ef41cbdc44 then I should receive 404 Not Found and the response body should contain: 
{
  "error" : {
    "message" : "No resource found with id 3ed4faaa-c0cd-11e4-a84e-83ef41cbdc44"
  }
}
How can I get all data for a given resource?
Given I have successfully /resources/6e56c868-c0a9-11e4-a762-cb9a664afc01 the following body: 
{
  "resource" : {
    "id" : "6e56c868-c0a9-11e4-a762-cb9a664afc01",
    "ref" : "http://www.example.com/titles/path/to/title/resource"
  }
}
When I /resources/6e56c868-c0a9-11e4-a762-cb9a664afc01 I should receive 200 OK with a response body containing 
{
  "resource" : {
    "id" : "6e56c868-c0a9-11e4-a762-cb9a664afc01",
    "ref" : "http://www.example.com/titles/path/to/title/resource",
    "^provenance" : "https://{host}/resources/6e56c868-c0a9-11e4-a762-cb9a664afc01/provenance"
  }
}
And /resources/6e56c868-c0a9-11e4-a762-cb9a664afc01/provenance should return 200 OK with a response body containing 
{
  "provenance" : {
    "who" : "nederlab",
    "why" : "",
    "when" : "{date.beforeNow}",
    "^what" : "https://{host}/resources/6e56c868-c0a9-11e4-a762-cb9a664afc01"
  }
}
