Creating external resources

This section explains how to register external resources to Alexandria via POST or PUT
The following use-cases are documented here:
  • Creating a resource via POST
  • Creating a resource via PUT
  • POSTing empty body
  • POSTing a body with no 'ref' field
  • Entity id must match id in URL

Examples:

How can I create an external resource without specifying an ID?
When I /resources the following body: 
{
  "resource" : {
    "ref" : "http://www.example.com/titles/path/to/title/resource"
  }
}
then I should receive 201 Created and the response contains a Location header starting with https://{host}/resources/, followed by a well-formed UUID.

What happens when I POST a body without a ref when creating a new resource?
When I /resources the following body: 
{
  "resource" : {
    "provenance" : {
      "who" : "nederlab",
      "why" : "some reason",
      "when" : "2015-04-07T09:10:55.877Z"
    }
  }
}
Then I should receive 400 Bad Request and the response body contains: 
[ {
  "message" : "Missing required 'ref' field"
} ]
When creating a new resource via PUT, the id in the entity body MUST match the one used in the URL
When I /resources/6e56c868-c0a9-11e4-a762-cb9a664afc01 the following body (note the mismatch for parameter id): 
{
  "resource" : {
    "id" : "3c86e050-f7d8-11e4-9c82-df9a2acb2bd1",
    "ref" : "http://www.example.com/titles/path/to/title/resource"
  }
}
Then I should receive 400 Bad Request and the response body contains: 
[ {
  "message" : "Entity 'id' must match the one used in the URL"
} ]
