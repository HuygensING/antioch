How do I submit a text as JSON and retrieve it as XML?
Given that I have successfully /resources/11111111-1111-1111-1111-111111111111 the body: 
{
  "resource" : {
    "id" : "11111111-1111-1111-1111-111111111111",
    "ref" : "grotius-3160-v1"
  }
}
And I have successfully /resources/11111111-1111-1111-1111-111111111111/baselayerdefinition the body: 
{
  "baseLayerDefinition" : {
    "subresourceElements" : [ "note" ],
    "baseElements" : [ {
      "name" : "text"
    }, {
      "name" : "p",
      "baseAttributes" : [ "auth", "lang" ]
    } ]
  }
}
using application/json, to set the base layer definition:

When I /resources/11111111-1111-1111-1111-111111111111/text the body: 
{
  "text" : {
    "body" : "<text><p xml:id=\"p-1\" lang=\"en\" auth=\"me\">paragraph 1</p><p>pârágràph 2</p></text>"
  }
}
using application/json Then I should receive 200 OK and the response body is 
{
  "resourceTextUploadResult" : {
    "^baseLayerDefinition" : "https://{host}/resources/11111111-1111-1111-1111-111111111111/baselayerdefinition"
  }
}

Thereafter, /resources/11111111-1111-1111-1111-111111111111/text using text/xml yields response body:
<text xml:id="text-1"><p xml:id="p-1" auth="me" lang="en">paragraph 1</p><p xml:id="p-2">pârágràph 2</p></text>
Notice that base layer elements have been given a unique xml:id if they didn't have one already.
Notice also that the attribute order on <p> is the one defined in the base layer definition.

What if I upload a text to a resource that already has a text?
Given the resource I made earlier, when I /resources/33333333-3333-3333-3333-333333333333/text the body: 
{
  "text" : {
    "body" : "<text><p>paragraph 1</p><p>paragraph 2</p><p>paragraph 3</p></text>"
  }
}
Then I should receive
  • status 409 Conflict
  • and body 
    {
  "error" : {
    "message" : "This resource already has a text, which cannot be replaced."
  }
}
What happens if no base layer is defined?
Given that I have successfully /resources/99999999-9999-9999-9999-999999999999 the body: 
{
  "resource" : {
    "id" : "99999999-9999-9999-9999-999999999999",
    "ref" : "grotius-3161-v2"
  }
}

When I then /resources/99999999-9999-9999-9999-999999999999/text the body: 
<text><p>the quick brown fox jumped the lazy dog's back</p></text>
using application/octet-stream,
Then I will receive 409 Conflict and the response body is 
{
  "error" : {
    "message" : "No base layer defined for resource: 99999999-9999-9999-9999-999999999999"
  }
}
