How can I create a tag?

Register the resource at Alexandria (once per resource)

  • We /resources/e480f748-3a9d-11e5-abc7-339c977c9f7e the body 
    {
  "resource" : {
    "ref" : "http://www.example.com/titles/path/to/title/resource",
    "provenance" : {
      "who" : "nederlab",
      "why" : "initial registration",
      "when" : "2015-08-04T13:46:39+02:00"
    }
  }
}
    and observe status 201 Created

Register and confirm the subresource at Alexandria (once per subresource)

  • To register, we /resources/e480f748-3a9d-11e5-abc7-339c977c9f7e/subresources the body 
    {
  "subresource" : {
    "sub" : "/some/folia/expression/marker"
  }
}
    and observe status 201 Created and a location header starting with https://{host}/resources/ and ending in a generated well-formed UUID for the newly created subresource.
    • In this case the generated uuid happened to be: 73b29203-3e7b-42b4-ab67-84e7980ddd04.
  • To confirm our subresource, we /resources/73b29203-3e7b-42b4-ab67-84e7980ddd04/state the body 
    {
  "state" : "CONFIRMED"
}
    and observe status 204 No Content
  • Duplicate submission of subresources is allowed
    Now that we know the subresource already exists, if we again /resources/e480f748-3a9d-11e5-abc7-339c977c9f7e/subresources the body 
    {
  "subresource" : {
    "sub" : "/some/folia/expression/marker"
  }
}
    then we receive status 204 No Content (instead of 201 Created) with the same location header: https://{host}/resources/73b29203-3e7b-42b4-ab67-84e7980ddd04

Register and confirm the annotation at the subresource.

  • To register, we /resources/73b29203-3e7b-42b4-ab67-84e7980ddd04/annotations the body 
    {
  "annotation" : {
    "type" : "emotion",
    "value" : "happy"
  }
}
    and observe status 201 Created
    • In this case we got uuid 30592a29-37a7-44f6-92da-1834f1fe2ad9 as the uuid for the annotation we just created.
  • (verification) We /annotations/30592a29-37a7-44f6-92da-1834f1fe2ad9 and observe status 200 OK and body: 
    {
  "annotation" : {
    "value" : "happy",
    "type" : "emotion",
    "id" : "30592a29-37a7-44f6-92da-1834f1fe2ad9",
    "state" : {
      "value" : "TENTATIVE",
      "since" : "{date.beforeNow}"
    },
    "^annotates" : "https://{host}/resources/73b29203-3e7b-42b4-ab67-84e7980ddd04",
    "^annotations" : [ ]
  }
}
    Note the TENTATIVE state of our newly created annotation.
  • To confirm, we /annotations/30592a29-37a7-44f6-92da-1834f1fe2ad9/state the body 
    {
  "state" : "CONFIRMED"
}
    and observe status 204 No Content
  • (verification) We /annotations/30592a29-37a7-44f6-92da-1834f1fe2ad9 and observe status 200 OK and body (only relevant fields shown for brevity) 
    {
  "annotation" : {
    "value" : "happy",
    "type" : "emotion",
    "id" : "30592a29-37a7-44f6-92da-1834f1fe2ad9",
    "^annotates" : "https://{host}/resources/73b29203-3e7b-42b4-ab67-84e7980ddd04"
  }
}
    Note that ^annotates indeed points to our subresource.
  • (verification) We /resources/73b29203-3e7b-42b4-ab67-84e7980ddd04 and observe status 200 OK and body (again, only relevant fields shown for brevity) 
    {
  "subresource" : {
    "id" : "73b29203-3e7b-42b4-ab67-84e7980ddd04",
    "^annotations" : [ "https://{host}/annotations/30592a29-37a7-44f6-92da-1834f1fe2ad9" ]
  }
}
    Note that ^annotations indeed includes our new annotation.

I. Get a (paged) overview of all tags owned by a user, sorted on timestamp

We show how to find all annotations of type Tag for user with id user-4242
We /searches the body 
{
  "query" : {
    "find" : "annotation",
    "where" : "who:eq(\"user-4242\") type:eq(\"Tag\")",
    "return" : "id,type,value,resource.url"
  }
}
and observe status 201 Created with the Location header pointing to the results of our query.
We follow the Location header and /searches/e2ad076d-3a03-4bff-9c55-b986f96324ca expecting 200 OK and body 
{
  "searchResult" : {
    "id" : "e2ad076d-3a03-4bff-9c55-b986f96324ca",
    "query" : {
      "find" : "annotation",
      "where" : "who:eq(\"user-4242\") type:eq(\"Tag\") state:eq(\"CONFIRMED\")",
      "sort" : "-when",
      "return" : "id,type,value,resource.url"
    },
    "totalResults" : 3,
    "totalPages" : 1,
    "^firstPage" : "https://{host}/searches/e2ad076d-3a03-4bff-9c55-b986f96324ca/pages/1"
  }
}
  • The sort field, which we left unspecified, defaults to "-when" (= reverse timestamp order: "most recent first").
  • The state of the where field, which we left unspecified, defaults to "CONFIRMED"
We /searches/e2ad076d-3a03-4bff-9c55-b986f96324ca/pages/1 expecting 200 OK and body
