How can I create a tag?

Register the resource at Alexandria (once per resource)

  • We /resources/e480f748-3a9d-11e5-abc7-339c977c9f7e the body 
    {
  "resource" : {
    "ref" : "http://www.example.com/titles/path/to/title/resource",
    "provenance" : {
      "who" : "nederlab",
      "why" : "initial registration",
      "when" : "2015-08-04T13:46:39+02:00"
    }
  }
}
    and observe status 201 Created

Register and confirm the subresource at Alexandria (once per subresource)

  • To register, we /resources/e480f748-3a9d-11e5-abc7-339c977c9f7e/subresources the body 
    {
  "subresource" : {
    "sub" : "/some/folia/expression/marker"
  }
}
    and observe status 201 Created and a location header starting with https://{host}/resources/ and ending in a generated well-formed UUID for the newly created subresource.
    • In this case the generated uuid happened to be: bc94b7f9-741b-4922-9924-4d69d3d3e7de.
  • To confirm our subresource, we /resources/bc94b7f9-741b-4922-9924-4d69d3d3e7de/state the body 
    {
  "state" : "CONFIRMED"
}
    and observe status 200 OK

Why do I get a 409 Conflict when using a newly created subresource?
The try-confirm-cancel pattern dictates we first confirm any newly created resources before using them.

Failure to follow this pattern leads to grief when operating on the resource!

Example: adding an annotation to an unconfirmed resource

  • Given that a resource with id 6244b9d8-3b5e-11e5-aca2-5befb33d5e4d exists and
  • a subresource is successfully registered via /resources/6244b9d8-3b5e-11e5-aca2-5befb33d5e4d/subresources with body: 
    {
  "subresource" : {
    "sub" : "/some/folia/path"
  }
}
    in this case yielding uuid c20dc4fe-c028-4c82-816d-50f272a4af2e.
  • When I /resources/c20dc4fe-c028-4c82-816d-50f272a4af2e/annotations an example annotation body:

    "type" : "t",

  • Then I will get status 409 Conflict and a response body containing:

    "message" : "resource c20dc4fe-c028-4c82-816d-50f272a4af2e is tentative, please confirm first"
  }
