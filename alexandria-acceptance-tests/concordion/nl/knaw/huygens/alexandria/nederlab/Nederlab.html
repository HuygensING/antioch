How can I create a tag?
  1. Register the resource at Alexandria, if this is the first use of the resource.
    We /resources/e480f748-3a9d-11e5-abc7-339c977c9f7e the following body 
    {
  "resource" : {
    "ref" : "http://www.example.com/titles/path/to/title/resource",
    "provenance" : {
      "who" : "nederlab",
      "why" : "initial registration",
      "when" : "2015-08-04T13:46:39+02:00"
    }
  }
}
    and observe status: 201 Created
  2. Register and confirm the subresource at Alexandria, if this is the first use.
    • To register, we
      /resources/e480f748-3a9d-11e5-abc7-339c977c9f7e/subresources the following body 
      {
  "subresource" : {
    "sub" : "/some/folia/expression/marker"
  }
}
      and observe status: 201 Created and a location header starting with https://{host}/resources/ and ending in the well-formed UUID of the newly created subresource.
      • In this case the generated uuid happened to be: 6e53f385-288a-464c-a37c-7c0714927847.
    • To confirm, we
      /resources/6e53f385-288a-464c-a37c-7c0714927847/state the following body 
      {
  "state" : "CONFIRMED"
}
      and observe status: 200 OK
  3. Register and confirm the annotation at the (confirmed) subresource.
    • To register, we
      /resources/6e53f385-288a-464c-a37c-7c0714927847/annotations the following body: 
      {
  "annotation" : {
    "type" : "emotion",
    "value" : "happy"
  }
}
      and observe status: 201 Created
      • In this case we got uuid 7e61a438-db4a-4cde-a896-662b710a4cad as the uuid for the annotation we just created.
    • (optional) As an intermediate verification step, we can /annotations/7e61a438-db4a-4cde-a896-662b710a4cad and observe status: 200 OK and body: 
      {
  "annotation" : {
    "value" : "happy",
    "type" : "emotion",
    "id" : "7e61a438-db4a-4cde-a896-662b710a4cad",
    "state" : {
      "value" : "TENTATIVE",
      "since" : "{date.beforeNow}"
    },
    "^annotates" : "https://{host}/resources/6e53f385-288a-464c-a37c-7c0714927847",
    "^annotations" : [ ]
  }
}
    • To confirm, we
      /annotations/7e61a438-db4a-4cde-a896-662b710a4cad/state the following body 
      {
  "state" : "CONFIRMED"
}
      and observe status: 200 OK
    • (optional) As a final verification, we can /annotations/7e61a438-db4a-4cde-a896-662b710a4cad and observe status: 200 OK and body: 
      {
  "annotation" : {
    "value" : "happy",

Why do I get a 409 Conflict when using a newly created subresource?
The try-confirm-cancel pattern dictates we first confirm any newly created resources before using them.

Failure to follow this pattern leads to grief when operating on the resource!

Example: adding an annotation to an unconfirmed resource

  • Given that a resource with id 6244b9d8-3b5e-11e5-aca2-5befb33d5e4d exists and
  • a subresource is successfully registered via /resources/6244b9d8-3b5e-11e5-aca2-5befb33d5e4d/subresources with body: 
    {
  "subresource" : {
    "sub" : "/some/folia/path"
  }
}
    in this case yielding uuid 53ca1624-9d56-4ecb-a166-b32851df92d6.
  • When I /resources/53ca1624-9d56-4ecb-a166-b32851df92d6/annotations an example annotation body: 
    {
  "annotation" : {
    "type" : "t",
    "value" : "v"
  }
}
  • Then I will get status 409 Conflict and a response body containing: 
    {
  "error" : {
    "message" : "resource 53ca1624-9d56-4ecb-a166-b32851df92d6 is tentative, please confirm first"
  }
}
