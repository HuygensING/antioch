1) Introduction
Antioch provides a (limited, for now) implementation of the W3C Web Annotation Protocol (W3C Candidate Recommendation 06 September 2016)
2) Create a New Annotation
When I /webannotations/ the following body: 
{
  "@context" : "http://www.w3.org/ns/anno.jsonld",
  "type" : "Annotation",
  "body" : {
    "type" : "TextualBody",
    "value" : "I like this page!"
  },
  "target" : "http://www.example.com/index.html"
}
using application/ld+json
and application/ld+json,
then I should receive:
  • status 201 Created
  • a Location header starting with https://{host}/webannotations/ and ending in a well-formed UUID
  • and the response body should contain: 
    {
  "@context" : "http://www.w3.org/ns/anno.jsonld",
  "type" : "Annotation",
  "body" : {
    "type" : "TextualBody",
    "value" : "I like this page!"
  },
  "target" : "http://www.example.com/index.html"
}
In this case we got uuid af620b42-b1ee-4eb2-baf1-e111f650848b as the uuid for the resource we just created.
3) Annotation Retrieval
Given the webannotation we made earlier, when I /webannotations/af620b42-b1ee-4eb2-baf1-e111f650848b
using application/ld+json; profile="http://www.w3.org/ns/anno.jsonld",
then I should receive
  • status 200 OK
  • and the response body should contain: 
    {
  "@context" : "http://www.w3.org/ns/anno.jsonld",
  "id" : "https://{host}/webannotations/af620b42-b1ee-4eb2-baf1-e111f650848b",
  "type" : "Annotation",
  "body" : {
    "type" : "TextualBody",
    "value" : "I like this page!"
  },
  "target" : "http://www.example.com/index.html"
}
4) Update an Exisiting Annotation
To change the webannotation we made earlier, I /webannotations/af620b42-b1ee-4eb2-baf1-e111f650848b the following body: 
{
  "@context" : "http://www.w3.org/ns/anno.jsonld",
  "type" : "Annotation",
  "body" : {
    "type" : "TextualBody",
    "value" : "On second thought, it's a boring page!"
  },
  "target" : "http://www.example.com/index.html"
}
using application/ld+json; profile="http://www.w3.org/ns/anno.jsonld"
and application/ld+json,
then I should receive status 200 OK
Now when I /webannotations/af620b42-b1ee-4eb2-baf1-e111f650848b
using application/ld+json; profile="http://www.w3.org/ns/anno.jsonld",
the response body should contain: 
{
  "@context" : "http://www.w3.org/ns/anno.jsonld",
  "id" : "https://{host}/webannotations/af620b42-b1ee-4eb2-baf1-e111f650848b",
  "type" : "Annotation",
  "body" : {
    "type" : "TextualBody",
    "value" : "On second thought, it's a boring page!"
  },
  "target" : "http://www.example.com/index.html"
}
5) Delete an Exisiting Annotation
When I /webannotations/af620b42-b1ee-4eb2-baf1-e111f650848b,
then I should receive status 204 No Content
After that, /webannotations/af620b42-b1ee-4eb2-baf1-e111f650848b
using application/ld+json; profile="http://www.w3.org/ns/anno.jsonld",
should give me status 410 Gone
6) Batch upload: AnnotationList (IIIF)
To add a batch of annotations, I can wrap them in an AnnotionList as defined by IIIF:
