1) What happens when I GET a non-existent resource?
Given that no resource exists with ID 3ed4faaa-c0cd-11e4-a84e-83ef41cbdc44
When I /resources/3ed4faaa-c0cd-11e4-a84e-83ef41cbdc44 then I should receive 404 Not Found and the response body should contain: 
{
  "error" : {
    "message" : "No resource found with id 3ed4faaa-c0cd-11e4-a84e-83ef41cbdc44"
  }
}
2) How can I get all data for a given resource?
Given I have successfully /resources/6e56c868-c0a9-11e4-a762-cb9a664afc01 the following body: 
{
  "resource" : {
    "id" : "6e56c868-c0a9-11e4-a762-cb9a664afc01",
    "ref" : "http://www.example.com/titles/path/to/title/resource"
  }
}
When I /resources/6e56c868-c0a9-11e4-a762-cb9a664afc01 I should receive 200 OK with a response body containing 
{
  "resource" : {
    "id" : "6e56c868-c0a9-11e4-a762-cb9a664afc01",
    "ref" : "http://www.example.com/titles/path/to/title/resource",
    "^provenance" : "https://{host}/resources/6e56c868-c0a9-11e4-a762-cb9a664afc01/provenance"
  }
}
And /resources/6e56c868-c0a9-11e4-a762-cb9a664afc01/provenance should return 200 OK with a response body containing 
{
  "provenance" : {
    "who" : "nederlab",
    "why" : "",
    "when" : "{date.beforeNow}",
    "^what" : "https://{host}/resources/6e56c868-c0a9-11e4-a762-cb9a664afc01"
  }
}

4) How can I find a resource with a given ref?
Given the following resource tree:
  • a (confirmed parent) resource exists with uuid: 00000000-0000-0000-0000-000000000004 and ref: Corpus
  • this resource has a confirmed subresource 1 with sub: Sub 1 and uuid: 941e67d5-78f5-468f-87dd-7312f164e879
  • and confirmed subresource 2, with sub: Sub 2 and uuid: 084f029b-882c-4c22-8335-c3c81ae3f2d0
  • which has its own confirmed subresource 2.1, with sub: Sub 2.1 and uuid: ac438a2f-cac6-49d4-813f-ea7da66779ce
  • and finally a different (confirmed parent) resource exists with uuid: 00000000-0000-0000-0000-000000000005 and ref: Corpus2
  • with confirmed subresource 3 with sub: Sub 2.1 and uuid: 684dc280-4ce1-4f19-b2f4-cd0afdbad8de
Now, to find the UUID of the resource in this tree that has ref Sub 2.1,
I /searches the following body 
{
  "query" : {
    "find" : "resource",
    "where" : "subresource.sub:eq(\"Sub 2.1\") resource.id:eq(\"00000000-0000-0000-0000-000000000004\")",
    "return" : "subresource.id"
  }
}
The uuid for this particular search was: 7e54c263-51de-4bb5-ae62-6c7522044590.
We /searches/7e54c263-51de-4bb5-ae62-6c7522044590 and observe 200 OK and the relevant subresource ids and subs in the body 
{
  "searchResultPage" : {
    "records" : [ {
      "subresource.id" : "ac438a2f-cac6-49d4-813f-ea7da66779ce"
    } ]
  }
}
