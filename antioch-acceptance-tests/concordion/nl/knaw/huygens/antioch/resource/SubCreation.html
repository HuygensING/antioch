Creating subresources

1) How can I add a subresource to a given resource?
Given a confirmed resource with id 00000000-0000-0000-0000-000000000001 exists.
When I /resources/00000000-0000-0000-0000-000000000001/subresources the body: 
{
  "subresource" : {
    "sub" : "/yellow/sub/marine"
  }
}
using application/json,
then I should receive
  • status 201 Created
  • and a location header starting with https://{host}/resources/ and ending in a generated well-formed UUID for the newly created subresource.
In this case the generated uuid happened to be: 59fc64cf-2f06-4431-ba2a-ff652d4cb37c.
To confirm our subresource, we /resources/59fc64cf-2f06-4431-ba2a-ff652d4cb37c/state the body 
{
  "state" : "CONFIRMED"
}
and observe status 204 No Content
2) How can I add a subresource with a given uuid to a given resource?
Given a confirmed resource with id 00000000-0000-0000-0000-000000000001 exists.
To add a subresource with 00000000-0000-0000-0000-000000000002,
I /resources/00000000-0000-0000-0000-000000000001/subresources/00000000-0000-0000-0000-000000000002 the body: 
{
  "subresource" : {
    "sub" : "some/sub/item"
  }
}
using application/json,
then I should receive
  • status 201 Created
  • and location header https://{host}/resources/00000000-0000-0000-0000-000000000002
To verify the newly created subresource, we
/resources/00000000-0000-0000-0000-000000000002 and observe status 200 OK
with body: 
{
  "subresource" : {
    "sub" : "some/sub/item",
    "id" : "00000000-0000-0000-0000-000000000002",
    "state" : {
      "value" : "CONFIRMED",
      "since" : "{date.beforeNow}"
    },
    "^partOf" : "https://{host}/resources/00000000-0000-0000-0000-000000000001",
    "^subresources" : [ ],
    "^annotations" : [ ],
    "^provenance" : "https://{host}/resources/00000000-0000-0000-0000-000000000002/provenance"
  }
}
Note that the subresource has state CONFIRMED, since we used a PUT to set it.
