1) What does a resource look like?
When we GET a resource, what does it look like and what is the meaning of the properties? Let us assume that:
  • a (parent) resource exists with uuid: 69f182ac-5c5f-11e5-8745-ef1581a11364
  • this resource has an annotation: 640bbba6-4ee7-4385-b392-36bacf39d6e6
  • this resource has a subresource: e1be327d-d3bb-4e5d-b9f0-6c54f1aae15e
  • this subresource has an annotation: 42530c1d-f48d-4a66-b11c-54defe91d946
When we /resources/69f182ac-5c5f-11e5-8745-ef1581a11364 we receive 200 OK and body 
{
  "resource" : {
    "id" : "69f182ac-5c5f-11e5-8745-ef1581a11364",
    "ref" : "http://www.example.com/some/ref",
    "state" : {
      "value" : "CONFIRMED",
      "since" : "{date.beforeNow}"
    },
    "^annotations" : [ "https://{host}/annotations/640bbba6-4ee7-4385-b392-36bacf39d6e6" ],
    "^provenance" : "https://{host}/resources/69f182ac-5c5f-11e5-8745-ef1581a11364/provenance",
    "^subresources" : [ "https://{host}/resources/e1be327d-d3bb-4e5d-b9f0-6c54f1aae15e" ]
  }
}
  • id shows the uuid of this resource
  • ref shows the external reference of this resource (which is where the actual contents this resource refers to can be found)
  • state.value is one of
    TENTATIVE
    the object has just been created, but not confirmed
    CONFIRMED
    the object has been confirmed and is in full operational order
    DELETED
    the object is marked as deleted
    DEPRECATED
    the object (usually an annotation) has been deprecated by a more recent version
  • state.since is the instant at which this resource's state was set to its current value
  • ^annotations contains a (possibly empty) array of links to the annotations on this resource
    Note that annotations of a subresource (i.c. 42530c1d-f48d-4a66-b11c-54defe91d946) are not shown in the ^annotations of the (parent) resource. (They can be fetched by explicitly GETting the subresource, or via the Query facility.)
  • ^provenance contains a link to the provenance object for this resource (who-what-when-why was this object created?)
  • ^subresources contains a (possibly empty) array of links to resources which are registered as being a subresource of this resource (in this case e1be327d-d3bb-4e5d-b9f0-6c54f1aae15e).
Subresources do not have a ref field, as they do not refer to an external location. Instead, they have a sub field which locates this subresource relative to its parent and a ^partOf link to that parent resource.
When we /resources/e1be327d-d3bb-4e5d-b9f0-6c54f1aae15e we receive 200 OK and body 
{
  "subresource" : {
    "id" : "e1be327d-d3bb-4e5d-b9f0-6c54f1aae15e",
    "sub" : "/some/folia/expression/0",
    "state" : {
      "value" : "CONFIRMED",
      "since" : "{date.beforeNow}"
    },
    "^partOf" : "https://{host}/resources/69f182ac-5c5f-11e5-8745-ef1581a11364",
    "^provenance" : "https://{host}/resources/e1be327d-d3bb-4e5d-b9f0-6c54f1aae15e/provenance",
    "^annotations" : [ "https://{host}/annotations/42530c1d-f48d-4a66-b11c-54defe91d946" ]
  }
}
