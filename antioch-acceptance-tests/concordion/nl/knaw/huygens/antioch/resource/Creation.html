2) How can I create an external resource without specifying an ID?
  • When I /resources the body: 
    {
  "resource" : {
    "ref" : "http://www.example.com/titles/path/to/title/resource"
  }
}
    then I should receive:
    • status 201 Created
    • a Location header starting with https://{host}/resources/ and ending in a well-formed UUID
    • an   response body
    • In this case we got uuid 7e9e1111-752a-4565-b54a-244944482411 as the uuid for the resource we just created.
  • (optional) To verify the newly created object, we /resources/7e9e1111-752a-4565-b54a-244944482411 and observe status 200 OK and body: 
    {
  "resource" : {
    "ref" : "http://www.example.com/titles/path/to/title/resource",
    "id" : "7e9e1111-752a-4565-b54a-244944482411",
    "state" : {
      "value" : "TENTATIVE",
      "since" : "{date.beforeNow}"
    },
    "^subresources" : [ ],
    "^provenance" : "https://{host}/resources/7e9e1111-752a-4565-b54a-244944482411/provenance",
    "^annotations" : [ ]
  }
}

5) When creating a new resource via PUT, the id in the entity body MUST match the one used in the URL

  }
}
Then I should receive 400 Bad Request and the response body contains: 
[ {
  "message" : "Entity 'id' must match the one used in the URL"
} ]
