1) How can I create a tag?

Register the resource at Antioch (once per resource)

  • We /resources/e480f748-3a9d-11e5-abc7-339c977c9f7e the body 
    {
  "resource" : {
    "ref" : "http://www.example.com/titles/path/to/title/resource",
    "provenance" : {
      "who" : "nederlab",
      "why" : "initial registration",
      "when" : "2015-08-04T13:46:39+02:00"
    }
  }
}
    and observe status 201 Created

Register and confirm the subresource at Antioch (once per subresource)

  • To register, we /resources/e480f748-3a9d-11e5-abc7-339c977c9f7e/subresources the body 
    {
  "subresource" : {
    "sub" : "/some/folia/expression/marker"
  }
}
    and observe status 201 Created and a location header starting with https://{host}/resources/ and ending in a generated well-formed UUID for the newly created subresource.
    • In this case the generated uuid happened to be: 9b9cb3bd-360b-4634-8094-56bf6540340f.
  • To confirm our subresource, we /resources/9b9cb3bd-360b-4634-8094-56bf6540340f/state the body 
    {
  "state" : "CONFIRMED"
}
    and observe status 204 No Content
  • Duplicate submission of subresources is allowed
    Now that we know the subresource already exists, if we again /resources/e480f748-3a9d-11e5-abc7-339c977c9f7e/subresources the body 
    {
  "subresource" : {
    "sub" : "/some/folia/expression/marker"
  }
}
    then we receive status 204 No Content (instead of 201 Created) with the same location header: https://{host}/resources/9b9cb3bd-360b-4634-8094-56bf6540340f

Register and confirm the annotation at the subresource.

  • To register, we /resources/9b9cb3bd-360b-4634-8094-56bf6540340f/annotations the body 
    {
  "annotation" : {
    "type" : "emotion",
    "value" : "happy"
  }
}
    and observe status 201 Created
    • In this case we got uuid 5d23390c-5768-4d12-9ce6-04b2d06da01a as the uuid for the annotation we just created.
  • (verification) We /annotations/5d23390c-5768-4d12-9ce6-04b2d06da01a and observe status 200 OK and body: 
    {
  "annotation" : {
    "value" : "happy",
    "type" : "emotion",
    "id" : "5d23390c-5768-4d12-9ce6-04b2d06da01a",
    "state" : {
      "value" : "TENTATIVE",
      "since" : "{date.beforeNow}"
    },
    "^annotates" : "https://{host}/resources/9b9cb3bd-360b-4634-8094-56bf6540340f",
    "^annotations" : [ ]
  }
}
    Note the TENTATIVE state of our newly created annotation.
  • To confirm, we /annotations/5d23390c-5768-4d12-9ce6-04b2d06da01a/state the body 
    {
  "state" : "CONFIRMED"
}
    and observe status 204 No Content
  • (verification) We /annotations/5d23390c-5768-4d12-9ce6-04b2d06da01a and observe status 200 OK and body (only relevant fields shown for brevity) 
    {
  "annotation" : {
    "value" : "happy",
    "type" : "emotion",
    "id" : "5d23390c-5768-4d12-9ce6-04b2d06da01a",
    "^annotates" : "https://{host}/resources/9b9cb3bd-360b-4634-8094-56bf6540340f"
  }
}
    Note that ^annotates indeed points to our subresource.
  • (verification) We /resources/9b9cb3bd-360b-4634-8094-56bf6540340f and observe status 200 OK and body (again, only relevant fields shown for brevity) 
    {
  "subresource" : {
    "id" : "9b9cb3bd-360b-4634-8094-56bf6540340f",
    "^annotations" : [ "https://{host}/annotations/5d23390c-5768-4d12-9ce6-04b2d06da01a" ]
  }
}
    Note that ^annotations indeed includes our new annotation.
2) How can I modify a tag?
Given a resource with id 13e53670-3f3f-11e5-96ef-dff0c5cf4b9b exists which has an annotation of type res-type and value res-value and which has the following annotated subresources
Subresource Id Subresource Sub Annotation type Annotation value
3f4ffa0c-3f4e-11e5-91d3-bfea30e2969c /some/folia/expression/0 subres1-type subres1-value
3570a6ca-3f4f-11e5-8a0d-4b7705ac2d85 /some/folia/expression/1 subres2-type subres2-value

Retrieve all annotations for all subresources of a given resource

We /searches the following body 
{
  "query" : {
    "find" : "annotation",
    "where" : "who:eq(\"nederlab\") resource.id:eq(\"13e53670-3f3f-11e5-96ef-dff0c5cf4b9b\")",
    "return" : "id,resource.url,subresource.url,subresource.sub,type,value"
  }
}
The uuid for this particular search was: e32e7edb-6b75-42b0-ab5a-1d41ba0adfd5.
We /searches/e32e7edb-6b75-42b0-ab5a-1d41ba0adfd5/pages/1 and observe 200 OK and the relevant resource and subresource urls in the body 
{
  "searchResultPage" : {
    "records" : [ {
      "type" : "res-type",
      "value" : "res-value",
      "resource.url" : "https://{host}/resources/13e53670-3f3f-11e5-96ef-dff0c5cf4b9b"
    }, {
      "type" : "subres1-type",
      "value" : "subres1-value",
      "resource.url" : "https://{host}/resources/13e53670-3f3f-11e5-96ef-dff0c5cf4b9b",
      "subresource.url" : "https://{host}/resources/3f4ffa0c-3f4e-11e5-91d3-bfea30e2969c",
      "subresource.sub" : "/some/folia/expression/0"
    }, {
      "type" : "subres2-type",
      "value" : "subres2-value",
      "resource.url" : "https://{host}/resources/13e53670-3f3f-11e5-96ef-dff0c5cf4b9b",
      "subresource.url" : "https://{host}/resources/3570a6ca-3f4f-11e5-8a0d-4b7705ac2d85",
      "subresource.sub" : "/some/folia/expression/1"
    } ]
  }
}

Update the annotation body

  • We /annotations/472f91a3-276d-4610-86b8-d9b0228fabac the body 
    {
  "annotation" : {
    "value" : "UPDATED-VALUE"
  }
}
    and observe status 204 No Content
  • (verification) We /annotations/472f91a3-276d-4610-86b8-d9b0228fabac and observe status 200 OK and the body 
    {
  "annotation" : {
    "id" : "472f91a3-276d-4610-86b8-d9b0228fabac",
    "type" : "res-type",
    "value" : "UPDATED-VALUE",
    "state" : {
      "value" : "CONFIRMED"
    },
    "revision" : 1,
    "^annotates" : "https://{host}/resources/13e53670-3f3f-11e5-96ef-dff0c5cf4b9b",
    "^deprecates" : "https://{host}/annotations/472f91a3-276d-4610-86b8-d9b0228fabac/rev/0",
    "^current_version" : "https://{host}/annotations/472f91a3-276d-4610-86b8-d9b0228fabac",
    "^versioned_self" : "https://{host}/annotations/472f91a3-276d-4610-86b8-d9b0228fabac/rev/1"
  }
}
    • The annotation value is updated, and its type is unchanged
    • The annotation state is CONFIRMED
    • The revision number has been incremented to 1
    • The new revision ^deprecates the previous one
    • Convenience links to both the ^current_version and to this particular revision ^versioned_self are included
  • (verification) We /resources/13e53670-3f3f-11e5-96ef-dff0c5cf4b9b and observe status 200 OK and the body 
    {
  "resource" : {
    "id" : "13e53670-3f3f-11e5-96ef-dff0c5cf4b9b",
    "^annotations" : [ "https://{host}/annotations/472f91a3-276d-4610-86b8-d9b0228fabac" ]
  }
}
    The resource still points to the same annotation (now containing the updated value).
  • (verification) We inspect the previous revision of the annotation by /annotations/472f91a3-276d-4610-86b8-d9b0228fabac/rev/0 and observe the body 
    {
  "annotation" : {
    "id" : "472f91a3-276d-4610-86b8-d9b0228fabac",
    "revision" : 0,
    "type" : "res-type",
    "value" : "res-value",
    "state" : {
      "value" : "DEPRECATED"
    }
  }
}
    The previous revision of the annotation has indeed become DEPRECATED.

Delete the annotation at its REST endpoint

  • Given a resource with id 349103a0-4006-11e5-8889-e3cb0d4e45c3 exists which is annotated.
  • (verification) We /resources/349103a0-4006-11e5-8889-e3cb0d4e45c3 and observe status 200 OK and body

    "id" : "349103a0-4006-11e5-8889-e3cb0d4e45c3",
    "^annotations" : [ "https://{host}/annotations/8656c289-a422-4c4f-aca1-fc5d7ac8c38e" ]
  }
}
  • We /annotations/8656c289-a422-4c4f-aca1-fc5d7ac8c38e and observe status 204 No Content and response body  
  • (verification) We /annotations/8656c289-a422-4c4f-aca1-fc5d7ac8c38e and observe status 200 OK and body

    "id" : "8656c289-a422-4c4f-aca1-fc5d7ac8c38e",
    "state" : {
      "value" : "DELETED",
      "since" : "{date.beforeNow}"

  }

    Note the DELETED state of the annotation.
  • (verification) We /resources/349103a0-4006-11e5-8889-e3cb0d4e45c3 and observe status 200 OK and body 
    {
  "resource" : {
    "id" : "349103a0-4006-11e5-8889-e3cb0d4e45c3",

I. Get a (paged) overview of all tags owned by a user, sorted on timestamp

We show how to find all annotations of type Tag for user with id user-4242
We /searches the body 
{
  "query" : {
    "find" : "annotation",
    "where" : "who:eq(\"user-4242\") type:eq(\"Tag\")",
    "return" : "id,type,value,resource.url"
  }
}
and observe status 201 Created with the Location header pointing to the results of our query.
 
We follow the Location header and /searches/ff0e12a5-dc1c-44ec-b93e-a19c5069b56d expecting 200 OK and body 
{
  "searchResultPage" : {
    "searchInfo" : {
      "id" : "ff0e12a5-dc1c-44ec-b93e-a19c5069b56d",
      "query" : {
        "find" : "annotation",
        "where" : "who:eq(\"user-4242\") type:eq(\"Tag\") state:eq(\"CONFIRMED\")",
        "sort" : "-when",
        "return" : "id,type,value,resource.url"
      },
      "totalResults" : 3,
      "totalPages" : 1
    },
    "^firstPage" : "https://{host}/searches/ff0e12a5-dc1c-44ec-b93e-a19c5069b56d/pages/1"
  }
}
  • The sort field, which we left unspecified, defaults to "-when" (= reverse timestamp order: "most recent first").
  • The state of the where field, which we left unspecified, defaults to "CONFIRMED"
We /searches/ff0e12a5-dc1c-44ec-b93e-a19c5069b56d/pages/1 expecting 200 OK and body 
{
  "searchResultPage" : {
    "pageNumber" : 1,
    "records" : [ {
      "id" : "{uuid.anyValid}",
      "type" : "Tag",
      "value" : "A more recent user comment.",
      "resource.url" : "https://{host}/resources/00000003-5b7f-11e5-898c-237ee1c6d1d4",
      "_resultNumber" : 1
    }, {
      "id" : "{uuid.anyValid}",
      "type" : "Tag",
      "value" : "An early user comment.",
      "resource.url" : "https://{host}/resources/00000002-3f4e-11e5-91d3-bfea30e2969c",
      "_resultNumber" : 2
    }, {
      "id" : "{uuid.anyValid}",
      "type" : "Tag",
      "value" : "A very early user comment.",
      "resource.url" : "https://{host}/resources/00000001-5ba6-11e5-9054-ab0dd746f7f7",
      "_resultNumber" : 3
    } ]
  }
}
